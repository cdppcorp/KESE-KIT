## 4. 핵심 개념/용어

### 점검 항목 코드 체계
- `U-XX`: Unix 서버
- `W-XX`: Windows 서버
- `WS-XX`: 웹 서비스
- `S-XX`: 보안 장비
- `N-XX`: 네트워크 장비
- `C-XX`: 제어시스템
- `PC-XX`: PC
- `D-XX`: DBMS
- `M-XX`: 이동통신
- `V-XX`: 가상화 장비
- `CL-XX`: 클라우드

### 공통 점검 영역
- 계정 관리 (모든 시스템)
- 접근 관리 / 접근 통제
- 패치 관리
- 로그 관리
- 서비스 관리 / 기능 관리

### 중요도 분류
- **상**: 침해 시 심각한 영향
- **중**: 침해 시 중간 수준 영향
- **하**: 침해 시 낮은 영향

## 7. KESE KIT 적용 관점

### 핵심 자동화 대상
1. **Unix/Windows 서버**: 스크립트 기반 점검 자동화
2. **네트워크 장비**: SSH/SNMP 기반 점검
3. **DBMS**: SQL 쿼리 기반 점검
4. **웹 서비스**: 설정 파일 점검 자동화

### 수동 점검 필요
1. **제어시스템**: OT 환경 특수성
2. **물리적 접근통제**: 현장 실사 필요
3. **가상화/클라우드**: 콘솔 접근 필요

### 우선순위 (바이브코딩 환경)
1. Unix/Linux 서버 점검 스크립트
2. 웹 서비스 설정 점검
3. DBMS 보안 점검
